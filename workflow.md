## Flujo de trabajo

Una vez tengas todo listo para empezar a editar el documento:

### 1. Cambia a la rama que se te asignó

(`marco_teorico`, `desarrollo`, `conclusiones`, etc.)

``` bash
git branch [TU_RAMA]
git push -u origin [TU_RAMA]
git checkout [TU_RAMA]
```

### 2. Realiza tu parte del trabajo

Edita los archivos correspondientes a tu sección.

### 3. Guarda y sube tus cambios a tu rama

``` bash
git add .
git commit -m "Se agregó la sección [TU_SECCION]"
git push
```
